# 学姐数据复现 CG/HydroCalculator 教学说明

## 0. 这一步先做什么

这一阶段不重新找数据、不替换参数，只用学姐已经整理好的输入表复现 HydroCalculator 的计算结果。

使用文件：

- 原始输入：`CG_data/泉水蒸发dateput.csv`
- 结果对照：`CG_data/泉水蒸发dateput_hydrocalc_output.csv`
- 扩展结果：`CG_data/result缙云山湖库泉20250927.csv`

生成文件：

- 复现结果：`results/senior_hydrocalc_reproduction.csv`
- 核对结果：`results/senior_hydrocalc_comparison.csv`
- 输入警告：`results/senior_hydrocalc_warnings.csv`
- 扩展画像：`results/senior_extended_reference_profile.csv`
- 摘要说明：`results/senior_hydrocalc_summary.md`

运行命令：

```powershell
python .\scripts\reproduce_senior_hydrocalc.py
```

## 1. 先认清输入表

| 字段 | 单位 | 在本数据中的角色 | 为什么需要 | 是否必须 |
|---|---:|---|---|---|
| `hydrocal=Sample name` | 文本 | 样品名或采样日期 | 追踪每一行结果对应哪个样品 | 是 |
| `opc` | 编码 | 计算模式 | 本表主要是 `3`，表示已知降水端元和 LEL | 是 |
| `T` | deg C | 温度 | 控制水-汽平衡分馏系数 | 是 |
| `h` | 0-1 | 相对湿度 | 控制蒸发富集强度，不能填百分数 | 是 |
| `d2HP` | per mil VSMOW | 初始水/入流水 δ2H | 代表蒸发前端元 | 是 |
| `d18OP` | per mil VSMOW | 初始水/入流水 δ18O | 与 δ2H 一起约束初始水 | 是 |
| `d2HL` | per mil VSMOW | 蒸发后水体 δ2H | 代表当前水体样品 | 是 |
| `d18OL` | per mil VSMOW | 蒸发后水体 δ18O | 判断水体蒸发富集程度 | 是 |
| `d2HA` | per mil VSMOW | 空气水汽 δ2H | `opc=3` 时可留空，由模型反推 | 否 |
| `d18OA` | per mil VSMOW | 空气水汽 δ18O | `opc=3` 时可留空，由模型反推 | 否 |
| `d2HR` | per mil VSMOW | 降水 δ2H | 用来估算空气水汽同位素 | 是 |
| `d18OR` | per mil VSMOW | 降水 δ18O | 用来估算空气水汽同位素 | 是 |
| `LEL` | 无量纲 | 本地蒸发线斜率 | 约束空气水汽反推过程 | 是 |
| `CkH` | per mil | δ2H 动力分馏常数 | 输入表缺省时用 12.5 | 否 |
| `CkO` | per mil | δ18O 动力分馏常数 | 输入表缺省时用 14.2 | 否 |

关键提醒：

- 同位素值统一理解为 per mil VSMOW。
- `h=0.72` 表示相对湿度 72%，不是 0.72%。
- 本表的 `d2HA/d18OA` 留空不是缺失错误，而是 `opc=3` 下由降水端元和 LEL 反推。

## 2. 单行复现：S01 2024.7.15

第一行输入：

| 字段 | 值 | 含义 |
|---|---:|---|
| `opc` | 3 | 已知降水端元 + LEL |
| `T` | 31.2 | 当日温度 |
| `h` | 0.72 | 当日相对湿度 |
| `d2HP/d18OP` | -79.83 / -11.67 | 蒸发前水体端元 |
| `d2HL/d18OL` | -38.5 / -6.17 | 当前水体样品 |
| `d2HR/d18OR` | -46.55 / -7.411 | 降水端元 |
| `LEL` | 4.5741 | 本地蒸发线斜率 |

这一行的教学意义：

- 当前水体 `d2HL/d18OL` 比初始水体 `d2HP/d18OP` 明显偏重，说明发生了蒸发富集。
- `opc=3` 会先根据降水端元和 LEL 搜索 `x`，再反推空气水汽 `d2HA/d18OA`。
- 学姐结果中这一行约为 `EI_H=0.3971`、`EI_O=0.2111`、`f_H=0.2461`、`f_O=0.1561`，可作为复现检查锚点。

## 3. 输出字段怎么读

| 字段 | 意义 | 论文中怎么说 |
|---|---|---|
| `aplusH/aplusO` | 平衡分馏系数 alpha+ | 由温度决定，温度越变，分馏系数随之改变 |
| `EplusH/EplusO` | 平衡富集量 epsilon+ | 表示水-汽相变过程的平衡同位素富集 |
| `EkH/EkO` | 动力富集量 epsilon_k | 由相对湿度和动力分馏常数决定 |
| `EH/EO` | 总富集项 | 综合平衡分馏和动力分馏 |
| `dstarH/dstarO` | 极限富集同位素值 | 用于计算蒸发损失的理论端点 |
| `mH/mO` | 蒸发富集曲线参数 | 控制 E/I 或 f 的计算 |
| `x` | LEL 匹配参数 | 模型为了让理论蒸发线接近输入 LEL 而搜索得到 |
| `d2HA/d18OA` | 空气水汽同位素 | 关键中间结果，影响蒸发损失估算 |
| `EI_H/EI_O` | 稳态蒸发/入流比 | 数值越大，稳态口径下蒸发相对越强 |
| `f_H/f_O` | 非稳态蒸发损失比例 | 数值越大，表示水体损失比例越高 |

论文表达时，不建议只报一个数字。更稳妥的写法是同时说明：

- 使用稳态还是非稳态口径。
- 使用 δ2H 还是 δ18O 结果，或使用二者均值。
- 参数来源是否一致：同一批次的温度、湿度、LEL、初始水和当前水体。

## 4. 批量核对怎么判断可信

核对文件 `results/senior_hydrocalc_comparison.csv` 中：

- `status=pass` 表示核心字段四位小数内与学姐结果一致。
- `max_abs_diff_core` 越接近 0，说明复现越好。
- 如果某行为 `check`，优先检查 `T`、`h`、`LEL`、`CkH/CkO` 和输入端元是否与学姐表一致。

输入警告文件 `results/senior_hydrocalc_warnings.csv` 中：

- 缺字段时不能直接删除点位，要记录原因。
- `h>1` 通常说明把百分数当成小数填入。
- `opc` 不是 3 时，需要单独解释计算模式。

## 5. 从模型结果转成论文意义

扩展画像文件 `results/senior_extended_reference_profile.csv` 用来服务这一部分。它不重新计算扩展表，只把学姐扩展结果里的水体类型、温湿度、LEL、`EI`、`f` 整理成更容易比较的格式。

可以从 5 个角度写：

1. 蒸发富集证据
   当前水体同位素比初始水体更偏重，说明水体经历蒸发富集。

2. 蒸发损失强度
   `EI` 或 `f` 越大，说明蒸发损失越明显。可以按日期、季节或水体类型比较。

3. δ2H 与 δ18O 的一致性
   如果 `EI_H` 和 `EI_O` 趋势一致，说明结果更稳；如果差异大，要讨论参数不确定性。

4. LEL 的水文意义
   LEL 是蒸发线，不是降水线。它反映水体蒸发过程，常与 LMWL/GMWL 对比来说明蒸发偏离程度。

5. 参数敏感性
   `h`、`T`、`LEL`、空气水汽 δA 都会影响结果，其中湿度和 δA 通常是重点不确定性来源。
